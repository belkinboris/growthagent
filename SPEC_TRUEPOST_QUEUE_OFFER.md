# SPEC: Блок «Собрать очередь на неделю» после good feedback

Статус: НЕ ВНЕДРЕНО. Внедряется только после того, как Борис нажмёт
«Принять» на рекомендации Growth Loop и передаст эту спецификацию
TruePost-чату (Claude).

## Зачем

Единственная правка TruePost для эксперимента `commercial_bridge`.
Гипотеза: платная ценность — регулярное ведение канала, а не один пост.
Пользователь получил хороший первый пост → сразу показать следующую
ценность (неделя контента) → мост к тарифам.

## Что сделать (ровно одно изменение)

В веб-приложении (static/app.js + main.py), в месте, где пользователь
нажимает «Нравится» на первом посте (то же место, где сейчас отправляется
`first_post_feedback` с package_id="good"):

1. После отправки good-feedback показать блок:

   Заголовок: «Соберём очередь на неделю?»
   Текст: «Автопост подготовит 7 постов по вашей теме — по одному на каждый
   день. Вы просто просматриваете и публикуете.»
   Под текстом: превью 3 тем ближайших постов (темы генерировать НЕ нужно —
   использовать существующую логику подбора тем канала; если её нет,
   показать статичный пример формата «Тема 1 / Тема 2 / Тема 3» из about
   канала — решает TruePost-чат по факту кода).
   Кнопка: «Собрать очередь» → ведёт на существующий экран тарифов.

2. Клик по кнопке логировать существующим механизмом:
   `POST /api/product-event` с event="pricing_viewed" срабатывает
   автоматически при открытии тарифов — дополнительно добавить в
   `_ALLOWED_PRODUCT_EVENTS` новое событие `queue_offer_clicked`
   (package_id пустой) и отправлять его при клике. Это позволит отличить
   «пришёл к тарифам через мост» от «пришёл сам».

3. Показ блока логировать событием `queue_offer_shown` (тоже добавить в
   allowlist) — чтобы считать конверсию показа, а не только клика.

## Что НЕ менять (locked variables эксперимента)

- generator.py и промпт первого поста
- тарифы, цены, состав тарифов
- onboarding до первого поста
- рекламу (Директ и Telegram Ads)
- лендинг

## Как измеряется результат

Growth Agent сам считает: `pricing_viewed / registrations` на новых
пользователях после старта эксперимента против baseline. Порог: 14 новых
регистраций или 14 дней. Вердикт автоматический. Новые события
queue_offer_shown/clicked дадут точную конверсию моста (P1: добавить их в
payment-path-diagnostics агрегаты).

## Откат

Скрыть блок (feature-флаг или удалить рендер). События в allowlist можно
оставить — они безвредны.

## Тесты (TruePost)

- good feedback → в ответе/состоянии появляется признак показа блока
- события queue_offer_shown/clicked принимаются product-event endpoint
- регрессия: test_onboarding_feedback.py проходит без изменений
