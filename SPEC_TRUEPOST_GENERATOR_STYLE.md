# ИНСТРУКЦИЯ ДЛЯ TRUEPOST-ЧАТА: итерация генератора под «не тот стиль»

Основание (данные за 7 дней, 07.07.2026): отзывов о первом посте 12,
из них 9 плохих (75%). Причины: «не тот стиль» ×3, «другое» ×3,
«не про тему» ×1, «слишком сухо» ×1. Запрет на generator.py снят —
условие «bad > 40%» выполнено. Это активный эксперимент Growth Agent
«Чиним качество первого поста»: вердикт по следующим 10 отзывам.

## Ключевой контекст, который объясняет «не тот стиль»

Генераций у НЕподключённых каналов: 30. У подключённых: 6.
То есть 5 из 6 первых постов генерируются, когда бот НЕ ВИДИТ реальный
канал пользователя — ему физически неоткуда взять стиль. Он пишет
«средним ИИ-тоном», и пользователь справедливо жмёт «не тот стиль».
Проблема не в качестве текста, а в отсутствии образца стиля.

## Что сделать (РОВНО одна итерация, ничего больше)

1. В момент первой генерации, если канал не подключён (unverified)
   и у пользователя нет style rules: перед генерацией предложить
   ОДИН необязательный шаг — «Вставьте 1–2 своих поста как образец
   стиля (можно пропустить)». Текст поля/кнопки — короткий, пропуск
   в один клик.

2. В промпт генерации добавить блок стилевого зеркалирования:
   - если образцы вставлены: «Проанализируй лексику, длину абзацев,
     обращение (ты/вы), эмодзи, ритм этих постов и напиши новый пост
     НЕОТЛИЧИМЫМ по стилю. Не копируй содержание.»
   - если образцов нет: строить стилевую рамку из полей онбординга
     (тема, описание, тип канала) и ЯВНО задавать один из 2–3
     пресетов тона по типу канала, а не «нейтрально».

3. Для подключённых каналов: убедиться, что существующий механизм
   (чтение заголовков/прошлых постов) реально подмешивает стиль в
   первый пост, а не только защищает от повторов тем.

## Что НЕ трогать (locked variables эксперимента)

- onboarding-шаги (кроме одного опционального поля из п.1)
- тарифы, цены, лендинг, реклама
- логику очереди, лимиты, количество регенераций
- структуру промпта, не относящуюся к стилю (антидефисы, структура
  абзацев — уже сделано, не переписывать)

## Как измеряется результат

Ничего не настраивать: Growth Agent сам считает долю good среди
СЛЕДУЮЩИХ 10 отзывов против baseline 25% (3/12) и выдаст вердикт.

## Откат

Вернуть прежний промпт-блок и скрыть поле образцов. Одна итерация =
один коммит, откат тривиален.

## Тесты

- unverified + образцы вставлены → в промпт попадает блок зеркалирования
- unverified + пропуск → в промпт попадает пресет тона по типу канала
- verified → стиль из реального канала подмешан
- регрессия: все существующие тесты TruePost проходят
