Обязательных поля три. Всё остальное аналитик определит сам: проверит связь, найдёт доступные endpoints и поставит стандартную разметку воронки. Проект должен реализовать внутренний API (минимум /api/internal/metrics c Bearer-токеном) — см. CONTRACT.md в репозитории.

Необязательно: внешние источники

OAuth-токены Яндекса задаются в переменных окружения сервера аналитика.

Чат с аналитиком

Аналитик отвечает по своим данным (воронка, оплата, эксперименты). Решений сам не принимает.
