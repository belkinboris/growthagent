# Экосистема: Создатель → Аналитик Воронки

## Позиционирование

Один путь «от идеи до денег», два инструмента, каждый на своей стадии:

| Стадия | Инструмент | Что происходит |
|---|---|---|
| ⓪ Оффер | **Создатель** | идея → 3 заострённых оффера |
| ① Спрос | **Создатель** | smoke-лендинг → реклама → события → вердикт по спросу |
| ② Продукт | (разработка) | вердикт положительный → делаем настоящий продукт |
| ③+ Рост | **Аналитик Воронки** | продукт живой → воронка, алерты, эксперименты, рост |

Точка передачи: продукт реализовал внутренний API
(`/api/internal/metrics` и далее по CONTRACT.md) и подключается к
Аналитику через мастер на вкладке «Проекты» — три поля (название,
base URL, токен), остальное автоматом. АвтоПост — первый проект,
прошедший этот путь.

Compass к этой связке отношения не имеет — это отдельный продукт.

## Как объединять (когда придёт время)

Оба сервиса — FastAPI на Timeweb, оба на Yandex LLM, оба однопользовательские
(Создатель — `SOZDATEL_OWNER_KEY`, Аналитик — `PLATFORM_ADMIN_PASSWORD`).
README Создателя сознательно требует раздельного деплоя («НЕ смешивать
с Аналитиком») — это правильно и сейчас, и после объединения фасада.

**Рекомендуемый вариант: один домен, два сервиса.** Аналитик вешается
на путь `/growth` домена Создателя (projectsozdatel.ru) через
реверс-прокси — вся платформа Аналитика уже живёт под этим префиксом:

```nginx
location /growth/ { proxy_pass http://growth-agent-app:8080; }
location /        { proxy_pass http://sozdatel-app:8080; }
```

Обычные пользователи Создателя `/growth` не видят: там форма входа
владельца, все API без сессии отвечают 401, страница с `noindex`.

**Связка продуктово, а не только на уровне домена:**

1. В кабинете Создателя (`/api/cabinet`, портфель со стадиями) карточка
   проекта на стадии ③+ ведёт на `/growth` — «проект передан Аналитику».
2. Обратно: Аналитик может показывать в обзоре ссылку «история проекта
   в Создателе» (idea_id из smoke-этапа).
3. Единый вход — позже: когда у Создателя появятся полноценные аккаунты
   (сейчас есть magic-link `/api/account/*`), проверка владельца Аналитика
   заменяется в одной точке (`app/platform_auth.py: require_admin`) на
   сессию Создателя.

**Вариант «одно приложение»** (include_router роутера платформы в main.py
Создателя) технически работает так же, как описывалось для любого
FastAPI-хоста, но пока не нужен: у сервисов разные БД, разные ритмы
деплоя, и падение одного не должно ронять второй.

## Деплой Аналитика: где угодно, не обязательно рядом с АвтоПостом

Аналитику нужно только: РФ-хостинг (из-за Yandex LLM и доступности
подключаемых продуктов), исходящий HTTPS к `base_url` подключённых
проектов и своя БД. «Рядом с АвтоПостом» было удобством первой
инсталляции, не требованием. Логичное место с учётом экосистемы —
рядом с Создателем (тот же Timeweb-аккаунт, домен projectsozdatel.ru,
путь /growth), потому что будущие проекты пользователя деплоиться
рядом не будут — они подключаются по HTTP откуда угодно.
