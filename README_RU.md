# Growth Agent Watchtower

Отдельный сервис, который смотрит на воронку digital-продукта (реклама →
регистрация → активация → оплата) и пишет в Telegram, где бизнес теряет
деньги. Первая версия — только наблюдение: бот ничего не меняет в рекламе,
продукте или оплатах, только анализирует и сообщает.

Первый подключённый проект — АвтоПост (TruePost). Сервис спроектирован как
универсальный: TruePost — первый адаптер (`connectors/truepost.py`), а не
основа архитектуры. Подробности — в `CONTRACT.md`.

## Что бот умеет в этой версии

- Каждые 3 часа (настраивается) собирает данные за три окна: 3 часа, 24 часа,
  7 дней.
- Сравнивает их с простыми правилами (например: "много кликов, но нет
  регистраций" или "оплаты начаты, но не завершены").
- Если находит проблему — пишет в Telegram один главный сигнал, без вороха
  цифр. Указывает confidence ("данных мало" / "проблема подтверждается").
- Не присылает одно и то же сообщение повторно — если проблема не изменилась,
  бот молчит. Если стало хуже — напомнит.
- Если сама интеграция (Метрика, Директ, TruePost) перестала отвечать —
  скажет отдельно, не путая это с проблемой в бизнесе.

## Что бот НЕ делает в этой версии

- Не меняет ставки и бюджеты в рекламе.
- Не меняет лендинг или продукт.
- Не пишет пользователям.
- Не трогает платежи.

Режимы `recommend_only`, `approval_required`, `autopilot_limited` заложены в
архитектуре на будущее, но в этой версии активен только `watch_only`.

## 2. Создание Telegram-бота

1. В Telegram напишите боту **@BotFather**.
2. Отправьте команду `/newbot`, придумайте имя и юзернейм бота.
3. BotFather даст токен вида `123456789:AAExampleTokenHere`. Это значение
   для `BOT_TOKEN` в `.env`.
4. Узнайте свой chat_id: напишите что угодно боту **@userinfobot**, он
   пришлёт ваш числовой ID. Это значение для `BOT_ADMIN_CHAT_IDS`
   (если несколько человек — через запятую).

---

## 3. Деплой на Railway

1. Загрузите этот репозиторий на GitHub (см. раздел 7 ниже, если нужна
   помощь с этим).
2. На [railway.app](https://railway.app) → **New Project** → **Deploy from
   GitHub repo** → выберите ваш репозиторий.
3. Railway сам определит Python-проект и применит `railway.toml` и
   `Procfile`.
4. Добавьте базу данных: в проекте Railway → **New** → **Database** →
   **PostgreSQL**. Railway автоматически создаст переменную `DATABASE_URL`
   и подставит её в ваш сервис — отдельно прописывать не нужно.
5. Перейдите в **Variables** вашего сервиса и добавьте остальные переменные
   из `.env.example` (см. раздел 4).
6. Railway передеплоит сервис автоматически после сохранения переменных.

## 4. Переменные окружения (.env)

Скопируйте `.env.example` и заполните:

| Переменная | Что это | Где взять |
|---|---|---|
| `DATABASE_URL` | Подключение к БД | Railway подставит сам для PostgreSQL; локально можно оставить `sqlite:///./growth_agent.db` |
| `BOT_TOKEN` | Токен Telegram-бота | От @BotFather, см. раздел 2 |
| `BOT_ADMIN_CHAT_IDS` | Ваш chat_id | От @userinfobot, см. раздел 2 |
| `PROJECT_NAME` | Название проекта для отображения | Например `АвтоПост` |
| `PROJECT_BASE_URL` | Адрес вашего продукта | Например `https://autopost26.up.railway.app` |
| `PROJECT_INTERNAL_API_TOKEN` | Секретный токен для связи с TruePost | Придумайте сами длинную случайную строку, впишите её же в TruePost (см. раздел 5) |
| `YANDEX_OAUTH_TOKEN` | OAuth-токен Яндекса | [oauth.yandex.ru](https://oauth.yandex.ru), нужен доступ к Метрике и Директу |
| `METRIKA_COUNTER_ID` | Номер счётчика Метрики | Из интерфейса Яндекс.Метрики |
| `DIRECT_CLIENT_LOGIN` | Логин в Яндекс.Директе | Ваш логин рекламного кабинета |
| `DIRECT_CAMPAIGN_IDS` | ID кампаний через запятую | Из интерфейса Директа |
| `WATCH_INTERVAL_SECONDS` | Как часто проверять | По умолчанию `10800` (3 часа) |

Если какая-то интеграция (Метрика, Директ, YooKassa) пока не настроена —
просто оставьте поле пустым. Сервис не упадёт, просто покажет эту
интеграцию как "не подключена" и не будет использовать её данные.

## 5. Добавление endpoint в TruePost

Growth Agent не имеет прямого доступа к базе данных TruePost — только по
HTTP. Нужно добавить один endpoint в сам TruePost.

1. Откройте `examples/truepost_internal_metrics_patch.py` — это готовый
   пример кода.
2. Скопируйте код в новый файл внутри проекта TruePost (например
   `internal_metrics.py`).
3. Замените заглушки на реальные запросы к вашей базе данных — в файле
   есть закомментированные примеры SQL-запросов через SQLModel, их нужно
   раскомментировать и подключить к вашим реальным моделям `User`,
   `Channel`, `Post`, `Payment`.
4. В главном файле приложения TruePost подключите роутер:
   ```python
   from internal_metrics import router as internal_metrics_router
   app.include_router(internal_metrics_router)
   ```
5. В переменных окружения TruePost добавьте:
   ```
   TRUEPOST_INTERNAL_API_TOKEN=придумайте-длинный-случайный-секрет
   ```
6. Тот же самый секрет впишите в Growth Agent как `PROJECT_INTERNAL_API_TOKEN`.
7. Передеплойте TruePost.

Важно: поле `as_of` в ответе обязательно. Это момент, на который посчитаны
данные. Без него Growth Agent откажется принимать ответ — это защита от
того, чтобы бот не считал устаревшие данные актуальными.

## 6. Проверка, что всё работает

После деплоя:

1. Откройте `https://ваш-growth-agent.up.railway.app/health` — должно
   вернуть `{"status": "ok"}`.
2. Откройте `https://ваш-growth-agent.up.railway.app/status` — должно
   показать ваш проект и статусы интеграций (скорее всего `not_configured`,
   если вы ещё не вписали все токены — это нормально).
3. В Telegram напишите вашему боту `/start` — должен ответить списком команд.
4. Напишите `/run` — бот запустит проверку прямо сейчас и покажет результат,
   даже если новых проблем не нашёл.
5. Если что-то не отвечает — проверьте логи в Railway (вкладка
   **Deployments** → последний деплой → **View Logs**).

### Команды бота

| Команда | Что делает |
|---|---|
| `/start` | Список команд |
| `/status` | Текущее состояние проекта и интеграций |
| `/run` | Запустить проверку вручную, показать результат |
| `/alerts` | Последние 10 сигналов |
| `/funnel` | Воронка за последние 7 дней |
| `/mode` | Текущий режим работы (`watch_only`) |
| `/settings` | Основные настройки проекта |
| `/debug_alert <id>` | Технические детали алерта (для отладки) |

## 7. Если вы впервые загружаете проект на GitHub

1. Создайте пустой репозиторий на [github.com](https://github.com) →
   **New repository** (без README и .gitignore — они уже есть здесь).
2. На своём компьютере, в папке с этим проектом:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЮЗЕРНЕЙМ/НАЗВАНИЕ.git
   git push -u origin main
   ```
3. Файл `.gitignore` уже настроен, чтобы не отправить на GitHub `.env` с
   вашими секретами — проверьте перед коммитом, что `.env` отсутствует в
   списке файлов (`git status`), только `.env.example`.

## Известные ограничения версии v1

- Поддерживается один активный проект (АвтоПост). Архитектура готова к
  нескольким, но интерфейс настройки для этого не сделан — подключение
  второго проекта потребует доработки.
- `connectors/metrika.py`, `connectors/direct.py`, `connectors/yookassa.py`
  — это интерфейсные заготовки (stub). Реальные вызовы API Яндекса и
  YooKassa нужно дописать перед использованием в проде.
- Нет автоматического подбора "нормы" метрик (baseline) — правила работают
  на фиксированных порогах, заданных в `config.py`.
- LLM не подключён — все объяснения в сообщениях бота собираются из готовых
  шаблонов в `rules.py`, без обращения к внешней модели.
